# 글로벌 의료 격차 완화 프로젝트 (SDGs 3·3.d 기반 건강 모니터링 웹앱)

## 📋 프로젝트 개요

이 웹앱은 **OECD 선진국과 개발도상국 간 의료 격차**를 분석하고,  
**저비용 디지털 헬스 기술(센서 + 모바일 앱)** 이 그 격차를 완화하는 하나의 실질적 방안이 될 수 있음을 제시하는 **사회문제 해결형 융합 프로젝트**입니다.

### 목적
- **UN SDGs 3번 목표 (건강과 웰빙)** 및 **3.d (보건 위험에 대한 조기 경보 및 관리 역량 강화)** 에 기반
- 국가 간 **의료인력·인프라·보건 지출·첨단 의료기술·원격진료·팬데믹 대응 역량**의 구조적 격차를 시각적으로 제시
- 저비용 아두이노 센서와 스마트폰만으로도 **기본 건강 데이터 측정 및 조기 경보 시스템**을 구현할 수 있음을 보여줌
- 개인 수준의 건강 데이터가 축적되면 **지역사회·국가 단위 공중보건 데이터**로 확장될 수 있음을 설명

### 기여
- 개인: 기본 바이탈 데이터의 **자가 모니터링과 위험 조기 인지** 지원  
- 지역사회: 위험 기록을 활용해 **지역 단위 건강 위험 패턴 파악**  
- 국가·국제사회: **데이터 기반 공중보건 정책 수립**과 **의료 격차 완화 전략** 설계에 기여

## 🧩 앱 전체 구조

앱은 단순한 도구가 아니라, **“문제 분석 → 데이터 기반 해결 가능성 제시”** 구조를 갖습니다.

- **홈 / 개요**  
  - 프로젝트 목적, SDGs 3·3.d 연결성 설명  
  - 의료 격차 현장을 보여주는 이미지 슬라이더(UNICEF 사진 기반)  

- **1. 글로벌 의료 격차 분석 페이지**  
  6가지 핵심 격차 영역을 카드 형식으로 제시:
  1. 의료인력·기초 인프라 격차  
  2. 보건의료 지출 및 R&D 투자 격차  
  3. 첨단 의료기술(AI·로봇수술·디지털헬스) 보급 격차  
  4. 원격진료(텔레메디슨) 활용의 양극화  
  5. 백신·의약품 생산 및 배분의 불평등  
  6. 감염병·팬데믹 대응 역량 차이  

- **2. 센서 기반 건강 모니터링 데모**  
  - 심박수·체온·산소포화도 등 **저비용 센서 예시**  
  - 실시간 그래프 + 정상/위험 범위 시각화  
  - “개인 건강 데이터 → 공중보건 데이터로 확장 가능성” 설명

- **3. 위험 기록 및 데이터 축적 화면**  
  - 위험 발생 시 자동 기록(시간·센서·값·위험 내용·정상 범위)  
  - 날짜 필터/상세 보기 + PDF 변환  
  - 데이터가 쌓일수록 **지역 보건 대응·조기 경보 시스템**의 기반이 됨을 설명

- **4. 정책·사회적 확장 설명 화면**  
  - 개발도상국·재난 지역·의료 취약 지역에서의 활용 시나리오  
  - 정부·국제기구·지역 보건 인력과 연계한 확산 전략 제시

## 🚀 주요 기능 (현재 웹앱 기준)

### 1. 글로벌 의료 격차 설명
- 6가지 격차 영역을 **카드 + 텍스트 + 실제 그래프/지도**로 구성
- 예시:
  - 인구 1,000명 당 의사 수 세계 지도  
  - OECD 국가별 **GDP 대비 보건의료 지출 비율** 막대 그래프  
  - 보건 직종 vs 기타 직종 **AI 관련 채용 공고 비중** 추이 그래프  
- 각 카드에서 **OECD vs 개발도상국**을 비교하고 데이터 출처(WHO, World Bank, OECD 등)를 명시

### 2. 실시간 센서 데이터 데모
- 아두이노 → 블루투스(HC-06) → Web Bluetooth API를 통해 데이터 수신 (또는 모의 데이터 모드)
- 센서별 목록 및 상세 페이지:
  - 현재 측정값(큰 숫자) 표시
  - Chart.js 기반 실시간 라인 그래프
  - 최대값 / 최소값 / 평균 자동 계산
  - 정상 범위 / 주의 / 위험 여부 판정
  - 위험 시 화면 상단에 **빨간 “위험 알림” 박스** 표시 및 자동 기록

### 3. 위험 기록 저장·관리
- 위험 상황 감지 시 다음 정보를 자동 저장:
  - 시간, 센서 종류, 측정값
  - 위험 내용(정상 범위 초과/미달 등)
  - 정상 범위(최소/최대 값)
- 기능:
  - 날짜별 필터
  - 리스트/상세 보기
  - 기록을 기반으로 **개인→지역→공중보건 단계**로 데이터가 확장되는 흐름 시각화

### 4. PDF 생성 기능
- pdfmake를 이용해 **위험 기록 리스트를 PDF 보고서로 자동 변환**
- PDF 구성:
  - 제목 및 생성 일시
  - 위험 기록 테이블(시간·센서·값·정상 범위·위험 내용)
  - 센서별 요약 통계(건수, 평균, 최소, 최대)
  - “이 문서는 SDGs 3.d 기반 조기 경보 및 관리 연구용”이라는 안내 문구

### 5. 응급 대처 및 정책 확장
- 홈 화면의 응급·정책 관련 카드:
  - 긴급 신고(장식용 버튼)
  - 심박수/체온/호흡 이상 및 일반 응급 상황에 대한 텍스트 가이드(아코디언 UI)
- 정책·사회적 확장 페이지:
  - 개발도상국, 재난 지역, 의료 취약 지역에서의 **구체적 활용 시나리오**
  - 기술 접근성·교육·데이터 보안·국제 협력 등 구현 전략

- **직관적 UI**: 파란색/화이트 기반의 깔끔한 의료 UI 스타일
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 모든 기기 지원
- **명확한 경고**: 위험 시 빨간색 박스로 즉시 알림

## 📱 사용 방법

### 1. 웹앱 실행
- `index.html` 파일을 브라우저에서 열기
- Chrome 또는 Edge 브라우저 권장 (Web Bluetooth API 지원)

### 2. 블루투스 연결
- 홈 화면에서 "연결하기" 버튼 클릭
- HC-06 블루투스 모듈 선택 후 페어링
- 연결 성공 시 실시간 센서 데이터 수신 시작

### 3. 센서 데이터 확인
- "센서 데모" 카드 클릭
- 센서 목록에서 원하는 센서(심박수/체온/산소포화도) 선택
- 실시간 값, 그래프, 통계 정보 확인

### 4. 위험 기록 확인
- "데이터 축적" 카드 클릭
- 날짜 필터로 특정 날짜 기록 확인
- 기록 클릭 시 상세 정보 확인
- "PDF 변환" 버튼으로 PDF 다운로드

### 5. 의료 격차 분석 / 정책 페이지
- "의료 격차 분석" 카드에서 6가지 격차 영역 설명/그래프 확인
- "정책 확장" 카드에서 개발도상국·재난·취약 지역 활용 시나리오 확인

## 🔧 개발 모드

블루투스 연결이 불가능한 환경에서는 자동으로 **모의 데이터 모드**가 활성화됩니다.
- 3초마다 랜덤한 센서 데이터 생성
- 모든 기능 테스트 가능

## 📊 지원 센서

현재 지원하는 센서:
- **심박수** (Heart Rate): 정상 범위 60-100 bpm
- **체온** (Temperature): 정상 범위 36.1-37.2°C
- **산소포화도** (Oxygen Saturation): 정상 범위 95-100%

## 🔒 데이터 저장

- 위험 기록은 **로컬 스토리지**에 저장됩니다
- 브라우저를 닫아도 기록이 유지됩니다
- PDF로 다운로드하여 영구 보관 가능

## 🌐 브라우저 호환성

- **Chrome/Edge**: 완전 지원 (Web Bluetooth API)
- **Firefox/Safari**: 제한적 지원 (모의 데이터 모드만 사용 가능)
